# Task Management API

## 📌 프로젝트 소개

Spring Boot 기반으로 업무(Task)를 관리하고,
상태 변경 이력을 함께 기록하여 데이터 추적이 가능하도록 설계한 API 서버입니다.

단순 CRUD 기능을 넘어,
업무 상태 변경 과정에서 발생하는 이력을 저장함으로써
운영 환경에서의 데이터 흐름을 추적할 수 있도록 구현했습니다.

---

## 🛠 기술 스택

* Java
* Spring Boot
* JPA (Hibernate)
* H2 Database
* JUnit5

---

## 🧩 주요 기능

* 업무(Task) 생성 / 조회 / 상태 변경 API
* Enum 기반 상태 관리 ('N', 'Y', 'E', 'C')
* 상태 변경 시 로그 자동 저장
* 사용자 IP 기반 요청 추적

---

## 🏗 아키텍처

* Controller → Service → Repository 구조
* 역할 분리를 통한 책임 기반 설계
* JPA 기반 객체 중심 데이터 처리

---

## 🔥 핵심 구현 포인트

* 상태 변경 시 이전 상태와 변경 상태를 함께 저장하여 이력 관리 기능 구현
* 로그에 사용자 IP를 저장하여 요청 단위 추적 가능
* @Transactional을 적용하여 데이터 정합성 보장
* Enum을 활용하여 상태 값의 안정성 확보

---

## 🧪 테스트

* 상태 변경 시 로그가 정상적으로 생성되는지 검증
* 존재하지 않는 데이터 요청 시 예외 처리 테스트

---

## 📂 API 예시

### 상태 변경 API

```http
POST /tasks/{sno}/status
```

```json
{
  "status": "Y"
}
```
